Estrategia anti-drift

La alineación ya no depende de recordar a mano qué puerto o qué README es el correcto. Queda respaldada por un manifiesto y por validación automatizada.

  1. `config/runtime/dashboard-reality.json` define superficies, estados permitidos y targets prohibidos para el dashboard raíz.
  2. `config/runtime/desktop-hub.json` sigue siendo la fuente de verdad del hub operativo del escritorio y sus launchers.
  3. `scripts/Validate-DashboardLinks.ps1` valida que no existan `href="#"`, que las rutas relativas resuelvan y que el dashboard no vuelva a apuntar a puertos legacy como entrada principal.
  4. Los cambios de runtime o de documentación deben actualizar primero los manifiestos y después las superficies HTML. Si el manifiesto no cambia, el dashboard tampoco debería cambiar.

Este dashboard es una vista alineada del repositorio. La operación cotidiana sigue entrando por `START_HERE.md`, el hub de escritorio generado y los scripts de `scripts/`.
